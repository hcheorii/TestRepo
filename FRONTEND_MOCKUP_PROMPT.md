# 📋 문서 관리 시스템 프론트엔드 목업 생성 프롬프트

## 🎯 생성형 AI에게 전달할 프롬프트

```
문서 관리 시스템의 프론트엔드 웹 애플리케이션 목업을 HTML과 CSS로 만들어주세요.

## 요구사항

### 📱 전체 레이아웃
- 모던하고 깔끔한 디자인
- 반응형 웹 디자인 (데스크톱 우선)
- 색상: 주요 색상은 파란색 계열, 흰색 배경, 회색 텍스트
- 폰트: 깔끔한 sans-serif 폰트 (예: Inter, Roboto)

### 🖥️ 전체 애플리케이션 구조
3개 섹션으로 구성:

1. **왼쪽 사이드바 (200px 고정 너비): 네비게이션**
   - 상단: "문서 관리" 로고/제목
   - 메뉴 항목:
     * 📄 "문서 목록" (1페이지 - 기본 선택)
     * 📁 "문서 관리" (2페이지)
   - 선택된 메뉴는 파란색 배경, 흰색 텍스트
   - 어두운 배경색 (#2D3748)

2. **메인 콘텐츠 영역 (나머지 너비)**

   **[1페이지] 문서 목록 페이지:**
   - 헤더: "전체 문서" 제목
   - 문서들을 카드 형태의 그리드로 표시 (4-5열)
   - 각 카드에는:
     * 파일 타입 아이콘 (PDF는 빨간색, 이미지는 파란색)
     * 파일명 (예: "프로젝트_보고서.pdf")
     * 파일 크기 (예: "2.5MB")
     * 업로드 날짜 (예: "2024.03.15")
     * 파일 확장자 표시
   - 하단에 페이지네이션 (1, 2, 3... 다음)
   - 카드 클릭 시 살짝 확대되는 호버 효과

   **[2페이지] 문서 관리 페이지:**
   - 2개 섹션으로 분할:
     * 왼쪽 (60%): 문서 그리드 (1페이지와 동일하지만 더 작은 카드)
     * 오른쪽 (40%): 미리보기 및 업로드 영역
       - 상단: 선택된 문서의 큰 이미지 미리보기
       - 중단: 선택된 파일 정보 (파일명, 크기, 날짜)
       - 하단: 파일 업로드 영역
         * "ADD" 버튼 (눈에 띄는 파란색)
         * 점선 테두리의 드래그 앤 드롭 박스
         * "파일을 드래그하거나 클릭하여 업로드" 텍스트
         * "업로드" 버튼

### 🎨 디자인 요소
- 카드는 그림자 효과 (shadow)
- 호버 시 카드가 살짝 올라오는 효과
- 선택된 카드는 파란색 테두리
- 버튼은 모서리가 둥근 형태
- 아이콘 사용 (파일 타입별, 업로드, 페이지네이션)

### 📄 샘플 데이터
다음과 같은 더미 데이터로 채워주세요:
- "프로젝트_보고서.pdf" (1.2MB, 2024.03.20)
- "회의_자료.jpg" (800KB, 2024.03.19)
- "데이터_분석.png" (1.5MB, 2024.03.18)
- "계약서.pdf" (2.1MB, 2024.03.17)
- "스크린샷.jpg" (950KB, 2024.03.16)
- "요구사항.pdf" (780KB, 2024.03.15)

### 💻 기술 요구사항
- 순수 HTML + CSS (JavaScript 없이)
- CSS Grid 또는 Flexbox 사용
- 반응형 디자인
- 모던 CSS 기법 (CSS Variables, hover effects)
- Font Awesome 또는 similar 아이콘 라이브러리 사용

### 🎯 주요 기능 UI
1. 문서 그리드의 카드 클릭 → 우측에 미리보기 표시되는 것처럼 스타일링
2. ADD 버튼은 눈에 띄게 (primary 색상)
3. 드래그앤드롭 영역은 점선 테두리와 아이콘
4. 페이지네이션은 하단 중앙 정렬

완성된 HTML 파일은 브라우저에서 바로 실행 가능해야 하며, 실제 서비스처럼 보이도록 완성도 높게 제작해주세요.
```

## 🎨 Figma/디자인 툴용 추가 프롬프트

```
문서 관리 시스템의 웹 애플리케이션 UI/UX 디자인을 만들어주세요.

## 화면 구성
- 데스크톱 웹 애플리케이션 (1440px 너비 기준)
- 2개 패널 레이아웃: 왼쪽 문서 그리드 + 오른쪽 미리보기/업로드

## 디자인 스타일
- 모던 SaaS 스타일
- 미니멀하고 깔끔한 디자인
- 색상: Primary Blue (#3B82F6), Gray (#6B7280), White (#FFFFFF)
- 타이포그래피: 깔끔한 sans-serif
- 카드 기반 UI
- 그림자와 호버 효과

## 주요 컴포넌트
1. 헤더: 제목 + ADD 버튼
2. 문서 카드 그리드 (3x2 레이아웃)
3. 페이지네이션
4. 이미지 미리보기 영역
5. 드래그앤드롭 업로드 영역

## 인터랙션 상태
- 기본 상태
- 카드 선택 상태 (파란색 테두리)
- 호버 상태
- 드래그 오버 상태

완성도 높은 프로덕션 레벨의 디자인으로 제작해주세요.
```

## 🤖 Claude/ChatGPT용 상세 프롬프트

```
사이드바 네비게이션이 있는 문서 관리 웹 애플리케이션의 프론트엔드 목업을 HTML + CSS로 제작해주세요.

## 📋 상세 요구사항




### 각 카드 정보 표시
- 파일 아이콘 (PDF는 빨간색, 이미지는 파란색)
- 파일명 (말줄임표 적용)
- 파일 크기
- 업로드 날짜
- 호버 시 shadow 효과

### 반응형 고려사항
- 모바일에서는 1열로 변경
- 우측 패널은 모바일에서 하단으로 이동

### 포함할 더미 데이터
6개 정도의 다양한 파일 (PDF, JPG, PNG 혼합)

완전히 동작하는 HTML 파일 하나로 제작해주세요. 모든 CSS는 <style> 태그 안에 포함하고, CDN에서 아이콘 라이브러리를 불러와 사용해주세요.
```
